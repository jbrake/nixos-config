# nixos-config

My NixOS config for Framework laptops.

Hosts:

| Host | Machine |
| --- | --- |
| `framework-amd-ai-300` | Framework 13, AMD Ryzen AI 9 HX 370 |
| `framework-intel-core-ultra` | Framework 13 Pro, Intel |

## What Lives Where

```text
flake.nix                              hosts, inputs, shared module wiring
flake.lock                             pinned nixpkgs/home-manager/etc.
hosts/<host>/configuration.nix         machine-specific kernel/GPU/touchpad bits
hosts/<host>/hardware-configuration.nix generated by nixos-generate-config
modules/nixos/base.nix                 users, networking, CLI tools, services
modules/nixos/containers.nix           Podman, Distrobox, DistroShelf
modules/nixos/desktop-plasma.nix       Plasma, SDDM, Steam, desktop apps
modules/nixos/fingerprint.nix          fprintd and fingerprint PAM behavior
home/jason/home.nix                    Home Manager user config
scripts/                               small rebuild/update/install helpers
```

## Install Notes

Back up the things Nix does not recreate:

- SSH keys
- browser profile
- Telegram data
- Calibre library
- PrismLauncher instances
- any local secrets under `/var/lib`, `~/.config`, or `~/.local/share`

### Normal Installer Flow

1. Boot the NixOS graphical ISO.

2. Run the installer:

```text
erase disk
no encryption
Plasma desktop
no swap
user: jason
```

This gives me a working desktop first. The real config replaces it after the
first reboot.

3. Clone this repo:

```bash
mkdir -p ~/Projects
cd ~/Projects
nix-shell -p git
git clone https://github.com/jbrake/nixos-config.git
cd nixos-config
```

4. Copy the generated hardware file into the right host:

```bash
cp /etc/nixos/hardware-configuration.nix hosts/framework-amd-ai-300/
```

5. Build it for next boot:

```bash
sudo NIX_CONFIG="experimental-features = nix-command flakes" \
  nixos-rebuild boot --flake .#framework-amd-ai-300
reboot
```

I use `boot` here because replacing the fresh desktop live is not worth it.

### Direct ISO Flow

For later, when I am comfortable doing the disk setup myself:

```bash
git clone https://github.com/jbrake/nixos-config.git
cd nixos-config
./scripts/install-host.sh framework-amd-ai-300
```

That script assumes the target system is already mounted at `/mnt`. It writes
the hardware config, locks the flake, installs NixOS, and copies the repo into
the new system.

This path uses the temporary `changeme` password from `base.nix`. Run `passwd`
right after the first login.

## Daily Commands

Apply current config:

```bash
sudo nixos-rebuild switch --flake .#framework-amd-ai-300
```

Shortcut:

```bash
./scripts/rebuild.sh framework-amd-ai-300
```

Add apps:

```text
GUI/desktop apps -> modules/nixos/desktop-plasma.nix
CLI/system tools -> modules/nixos/base.nix
user/dev tools  -> home/jason/home.nix
```

## Updating

Preview/update flow:

```bash
nix flake update
nix build .#nixosConfigurations.framework-amd-ai-300.config.system.build.toplevel --no-link
sudo nixos-rebuild switch --flake .#framework-amd-ai-300
```

Commit `flake.lock` only after the system seems fine.

If an update is bad:

```bash
git restore flake.lock
```

Then reboot and pick the previous generation from the boot menu if needed.

Firmware is separate:

```bash
fwupdmgr refresh
fwupdmgr update
```

Flatpaks are separate too:

```bash
flatpak update
```

Cleanup:

```bash
sudo nix-collect-garbage --delete-older-than 14d
```

## Other Desktops

Do not test another desktop from my real account. Desktop environments write
state into `$HOME`, and NixOS does not stop that.

Safer pattern:

```nix
users.users.detest = {
  isNormalUser = true;
  initialPassword = "detest";
};
```

Enable the desktop temporarily, rebuild, log in as `detest`, then remove the
desktop and test user when done.

## Fingerprint Notes

Fingerprint auth is sudo-only on purpose.

Reason: PAM fingerprint auth can block normal password prompts while it waits
for a scan. Sudo is fine because the terminal is already waiting. Login,
SDDM, and polkit stay password-first.

The Plasma lock screen uses KDE's own fingerprint path through fprintd, so it
does not need the same PAM setup.

Useful commands:

```bash
fprintd-list jason
fprintd-verify jason
fprintd-delete jason
```

Fingerprint enrollments live in `/var/lib/fprint`, not in this repo.

## Public Repo Notes

Never commit:

- SSH keys
- Tailscale auth keys
- VPN configs or tokens
- browser profile data
- app login state
- decrypted secrets

Stuff that is public here by design:

- hostnames
- hardware model notes
- disk UUIDs in generated hardware config
- package list
- my Git identity

Older pushed commit metadata is left as-is. Cleaning it up would require
rewriting history and force-pushing.
